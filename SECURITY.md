# Security Policy

This repository currently contains educational documentation rather than an
executable application.

## Report Privately

Use GitHub private vulnerability reporting if it is enabled. Otherwise contact
the repository owner through their GitHub profile.

Report privately when an issue involves:

- exposed credentials or personal information;
- malicious links or downloads;
- workflow changes that could compromise contributors;
- prompt examples that encourage unsafe destructive operations;
- supply-chain or GitHub Actions vulnerabilities.

For ordinary corrections, attribution concerns, and broken links, open a public
issue.

Do not include active secrets, personal data, or exploit instructions in a
public report.

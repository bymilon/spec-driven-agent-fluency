# Spec-Driven Agent Fluency

An open study guide for learning spec-driven development, professional
engineering English, and reliable workflows with AI coding agents.

The repository converts agent-workflow concepts into active-recall drills,
production-oriented prompt patterns, vocabulary exercises, and practical
review checklists.

## Who This Is For

- Engineers learning to work effectively with coding agents
- Non-native English speakers developing professional technical fluency
- Interns and junior engineers learning specification, validation, and review
- Teams discussing repeatable AI-assisted development workflows

## Start Here

1. Read [AI Agent Fluency Playbook](AI_AGENT_FLUENCY_PLAYBOOK.md).
2. Study one section of
   [Spec-Driven Development Drills](SPEC_DRIVEN_DEVELOPMENT_DRILLS.md).
3. Use the [Prompt Library](prompts/README.md) on a real repository task.
4. Answer each recall drill without looking at the lesson.
5. Rewrite one prompt from your actual work.
6. Review it after 1, 3, 7, 14, and 30 days.

## Core Workflow

```text
Discover
-> Specify
-> Clarify
-> Plan
-> Implement
-> Validate
-> Review
-> Integrate
-> Learn
```

Every strong agent task should identify:

- the objective;
- authoritative context;
- required artifacts;
- unresolved decisions;
- operational constraints;
- observable validation criteria.

## Learning Method

The exercises emphasize:

- **retrieval practice:** recalling an answer before reviewing it;
- **spaced practice:** revisiting material over increasing intervals;
- **interleaving:** mixing specification, testing, review, and language skills;
- **generation:** writing your own prompt before reading an example;
- **feedback:** comparing your work against explicit criteria.

These methods support durable learning. This project does not claim to diagnose,
treat, or medically improve neurological conditions. "Neuroplasticity" is not
used here as a promise of accelerated brain change.

## Suggested Weekly Routine

| Day | Practice |
| --- | --- |
| Monday | Study one lesson and complete its recall drill |
| Tuesday | Rewrite a vague engineering request |
| Wednesday | Practice a spoken status update |
| Thursday | Write one evidence-based review finding |
| Friday | Run a mixed scenario without notes |
| Weekend | Review errors and schedule spaced repetitions |

## Quality Standard

Examples in this repository aim to be production-oriented, but no generic
prompt is automatically production-ready. Adapt every instruction to the
repository's architecture, risk, security policy, compliance obligations, and
validation pipeline.

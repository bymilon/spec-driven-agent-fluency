# Repository Agent Instructions

## Purpose

Maintain an accessible, evidence-aware study guide for spec-driven development,
professional engineering English, and AI coding-agent workflows.

## Authoritative Files

- `README.md` defines project scope and public claims.
- `THIRD_PARTY_NOTICES.md` defines attribution and redistribution boundaries.
- `CONTRIBUTING.md` defines contribution requirements.
- `TODO.md` tracks release and learning improvements.

## Required Practices

1. Write original material. Do not reproduce unlicensed course prompts,
   graphics, books, or paid educational content.
2. Keep `resources/` untracked unless written redistribution permission is
   documented.
3. Cite first-party documentation, standards, or primary research for factual
   claims.
4. Distinguish observed evidence, inference, and recommendation.
5. Do not claim that exercises diagnose, treat, or guarantee cognitive or
   neurological improvement.
6. Use plain professional English suitable for non-native speakers.
7. Keep examples concise, testable, and explicit about constraints and
   validation.

## Validation

Before completing documentation changes:

1. Check relative links.
2. Check Markdown headings and tables.
3. Search for copied or unattributed third-party material.
4. Confirm `resources/` is not staged.
5. Summarize sources, checks, and remaining uncertainty.
